This document follows the spirit of the [Developer Certificate of Origin (DCO)](https://developercertificate.org/) — lightweight, no copyright assignment, no exclusivity.

## By contributing, you certify that:

1. **The contribution is your own original work**, OR you are submitting work that you have the right to submit under the AGPL-3.0-or-later, and you have clearly attributed any third-party code or content included.
2. **You agree that your contribution is licensed under the AGPL-3.0-or-later**, the same license as the rest of the project. You keep the copyright of your contribution. There is no copyright transfer — Xpendia Labs only receives the same AGPL license that every other user of the code receives.
3. **You have the right to submit it.** If you work for a company, you have the relevant permission (employment contract / employer waiver / open-source policy) to contribute this code to a public open-source project.
4. **Your contribution is provided "AS IS"**, without warranties of any kind, as is standard for open-source contributions.

That's it. No CLA gatekeeping, no signature ceremony.

## How to certify

Simply **open a pull request**. By submitting it, you accept the terms above.

If you want an explicit paper trail, add the following line to the body of your PR (or to your commit message):

```
Signed-off-by: Your Name <your@email.com>
```

This is the standard DCO sign-off line and is supported by the `git commit -s` flag.

## What we will NOT do

- **We will not relicense your contribution under a proprietary or commercial license.** The project is and will stay AGPL-3.0. If we ever change that, it would only be to a license at least as permissive (e.g., LGPL or MIT), never more restrictive.
- **We will not transfer your copyright.** You own what you wrote. We license it under AGPL like everyone else.
- **We will not sell exceptions.** If a downstream user needs a non-AGPL license, that's between them and the original Strudel maintainers — Xpendia Labs doesn't sell license escapes.
